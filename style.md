# FITCORE Design System Style Guide

> 适用范围：健身 App「今日」页及其相关训练、恢复、身体状态、营养记录模块。  
> 目标：同一套视觉语言同时覆盖日间模式与夜间模式，保持“科学训练、持续进步”的专业感、轻量感和可执行感。

---

## 1. 设计定位

本设计系统采用「轻背景 + 高对比移动端卡片」的产品展示风格。整体应避免传统健身 App 的重金属、强烈肌肉宣传感，而应更接近“数据化训练工具”：清晰、克制、科学、可长期使用。

视觉关键词：

- 科学训练
- 周期计划
- 恢复管理
- 营养对账
- 数据追踪
- 克制、专业、现代、轻盈

核心视觉特征：

- 大面积留白或深色空白，避免信息拥挤。
- 卡片化信息层级，所有模块以圆角容器承载。
- 使用高亮黄绿色作为唯一主品牌色。
- 日间模式保持干净、柔和、低噪声。
- 夜间模式不是纯黑堆叠，而是深石墨色、深蓝黑、深灰绿的层次组合。
- 训练图片只用于核心训练卡，不宜滥用。
- 数据模块使用线性进度条、小型指标卡、状态标签，不使用复杂大图表。

---

## 2. 画布与设备规格

### 2.1 移动端基础画布

推荐以 iPhone 竖屏尺寸为基准：

```text
设计稿基准宽度：390px
设计稿基准高度：844px
比例：9:16 / 近似移动端标准视口
安全区：顶部 44px，底部 34px
内容左右边距：20px
```

### 2.2 单手操作原则

主要操作按钮应放在屏幕中下部，避免只放在顶部。

优先级：

1. 主要训练操作：`开始今日训练`
2. 今日处理操作：`跳过今天（顺延）`、`想加练？`
3. 状态操作：`保存`、`记录`、`查看趋势`
4. 长周期操作：`从下周一开启`、`立即减载`、`预设下次训练为减载期`

## 3. 颜色系统

### 3.1 品牌主色

主色为偏黄的青绿色 / 柠檬绿，不使用纯绿色。它既承担品牌识别，也承担按钮、选中态、进度、正向反馈。

```css
--brand-lime-50:  #F4FFD2;
--brand-lime-100: #E9FF9A;
--brand-lime-300: #D6F653;
--brand-lime-500: #B8E81E;
--brand-lime-600: #9FD116;
--brand-lime-700: #7DAA10;
```

推荐主色：

```css
--color-primary: #B8E81E;
```

日间模式可略亮：

```css
--color-primary-light: #C9EA31;
```

夜间模式可略压暗但保持发光感：

```css
--color-primary-dark: #A9D91D;
```

### 3.2 日间模式颜色

```css
--light-bg-page:       #F7F8F2;
--light-bg-surface:    #FFFFFF;
--light-bg-subtle:     #F4F6EF;
--light-bg-tint:       #F1F9D8;

--light-text-primary:  #1B1D1F;
--light-text-secondary:#5F6468;
--light-text-tertiary: #9AA0A6;
--light-text-disabled: #B8BDC2;

--light-border:        #E6E9DF;
--light-border-strong: #D6DEC9;

--light-shadow:        0 12px 30px rgba(20, 28, 20, 0.08);
--light-shadow-card:   0 4px 14px rgba(20, 28, 20, 0.06);
```

### 3.3 夜间模式颜色

夜间模式不要使用纯黑作为主底色。推荐采用深蓝黑 / 深石墨色，形成更柔和的暗色层级。

```css
--dark-bg-page:        #0F1416;
--dark-bg-surface:     #151B1E;
--dark-bg-card:        #1B2225;
--dark-bg-card-elevated:#20282C;
--dark-bg-subtle:      #273034;

--dark-text-primary:   #F5F7F8;
--dark-text-secondary: #B6BEC4;
--dark-text-tertiary:  #7E878D;
--dark-text-disabled:  #555F66;

--dark-border:         #2B3438;
--dark-border-strong:  #3A464B;

--dark-shadow:         0 16px 34px rgba(0, 0, 0, 0.36);
--dark-shadow-card:    0 8px 20px rgba(0, 0, 0, 0.28);
```

### 3.4 功能色

```css
--color-success: #75D64B;
--color-warning: #F5A623;
--color-danger:  #FF6B4A;
--color-info:    #5AA7FF;

--color-protein: #63D36B;
--color-carb:    #F0C84B;
--color-fat:     #FF7A45;
```

使用规则：

- 训练完成、达标、正向趋势：使用 `--color-success` 或 `--color-primary`。
- 减载提醒：使用浅黄绿 / 轻警示，不要过度紧张。
- 过度训练预警：使用橙色，但面积要小，避免破坏主视觉。
- 休息日提示：可使用低饱和蓝色，表达恢复、冷静。
- 营养超标：使用橙色或红橙色，不要使用纯红。

## 4. 字体系统

### 4.1 字体族

中文优先使用系统字体，保证移动端渲染稳定。

```css
font-family:
  -apple-system,
  BlinkMacSystemFont,
  "SF Pro Display",
  "SF Pro Text",
  "PingFang SC",
  "Hiragino Sans GB",
  "Microsoft YaHei",
  "Noto Sans CJK SC",
  sans-serif;
```

如需更强品牌感，可在标题或数字中使用：

```css
font-family: "SF Pro Display", "Inter", "DIN Alternate", "PingFang SC", sans-serif;
```

### 4.2 字号层级

以 390px 宽移动端为基准。

```text
Page Title / 今日：             28px / 34px / 700
Section Title：                 17px / 24px / 700
Card Title：                    20px / 28px / 700
Metric Number：                 24px / 30px / 700
Metric Number Small：           18px / 24px / 700
Body Text：                     14px / 21px / 400
Secondary Text：                12px / 18px / 400
Caption / Meta：                11px / 16px / 400
Button Text：                   15px / 22px / 700
Tab / Chip Text：               12px / 18px / 600
```

### 4.3 数字排版

数据是本 App 的核心信息，应保证数字清晰。

要求：

- 数字可比普通文字略大一级。
- 数字与单位拆分层级，例如 `72.6` 使用 20–24px，`kg` 使用 11–12px。
- 百分比、进度、训练量、热量等关键数字应使用等宽数字特性：

```css
font-variant-numeric: tabular-nums;
```

## 5. 间距系统

采用 4px 基础网格。

```css
--space-2:  2px;
--space-4:  4px;
--space-6:  6px;
--space-8:  8px;
--space-10: 10px;
--space-12: 12px;
--space-16: 16px;
--space-20: 20px;
--space-24: 24px;
--space-32: 32px;
```

### 5.1 页面边距

```text
页面左右边距：20px
模块之间距离：16px
卡片内部边距：14–16px
紧凑卡片内部边距：12px
大按钮上下边距：14px
底部导航高度：72–84px
```

### 5.2 信息密度

该设计信息量较大，但必须保持“看起来轻”。控制方法：

- 每屏最多 4 个主模块。
- 每个模块标题与内容距离 8px。
- 模块与模块之间距离 14–16px。
- 指标卡之间间距 8px。
- 长文本不超过两行。
- 次级说明文字降低对比度，不使用粗体。

## 6. 圆角与边框

整体使用大圆角，但不要过度“气泡化”。

```css
--radius-xs:  6px;
--radius-sm:  10px;
--radius-md:  14px;
--radius-lg:  18px;
--radius-xl:  24px;
--radius-pill: 999px;
```

使用规则：

```text
页面主卡片：16–20px
训练图片卡：18–22px
按钮：14–999px，根据按钮形态决定
小标签 / Chip：999px
输入框：10–12px
底部导航容器：24px
手机外框：设备 mockup 自带圆角
```

边框：

```css
border: 1px solid var(--light-border);
border: 1px solid var(--dark-border);
```

夜间模式下边框比阴影更重要，卡片边界主要靠深浅差 + 细边框建立。

## 7. 阴影与层级

### 7.1 日间模式

日间模式使用轻阴影，避免厚重拟物。

```css
box-shadow: 0 4px 14px rgba(20, 28, 20, 0.06);
```

核心卡片可使用：

```css
box-shadow: 0 12px 30px rgba(20, 28, 20, 0.08);
```

### 7.2 夜间模式

夜间模式以表面层级为主，阴影为辅。

```css
box-shadow: 0 8px 20px rgba(0, 0, 0, 0.28);
```

不建议使用大面积绿色外发光。只有以下位置可使用轻微 glow：

- 当前选中日期
- 主 CTA
- 完成状态大勾
- 当前底部导航图标

```css
box-shadow: 0 0 18px rgba(184, 232, 30, 0.22);
```

## 8. 页面结构

「今日」页固定为自上而下结构：

```text
1. 顶部状态栏 / 安全区
2. 页眉：日期、星期、今日、计划切换
3. 日期横条或状态标签
4. 条件触发提示栏
5. 核心训练课表模块
6. 身体状态打卡模块
7. 饮食营养对账模块
8. 底部导航
```

### 8.1 页眉

内容：

- 日期：`5月14日  星期三`
- 页面标题：`今日`
- 计划切换：`训练日 ▾` 或 `力量增肌计划 ▾`
- 可选：提醒图标、计划日历按钮

样式：

```text
标题“今日”：28px，700
日期：12px，400，次级色
计划切换：胶囊按钮，高度 32px，圆角 999px
```

## 9. 组件规范

### 9.1 条件提示栏

#### 减载提示

文案示例：

```text
减载期提醒
本周为减载期，建议降低训练强度与总量，保持恢复优先。
```

视觉：

- 日间：浅黄绿背景，黄绿色图标。
- 夜间：深橄榄绿背景，绿色描边。
- 按钮：`了解更多`、`从下周一开启`。

#### 过度训练预警

文案示例：

```text
过度训练预警
连续8周未减载，注意中枢神经疲劳风险。
```

视觉：

- 使用橙色图标。
- 背景为低透明橙色，不使用大面积红色。
- CTA：`立即减载`、`查看建议`。

---

### 9.2 核心训练卡片

训练日状态 A 的卡片应为页面视觉重点。

内容结构：

```text
今日训练计划
[训练图片 / 动作视觉]
Day A1 下肢力量
6 个动作 · 14 组 · 中等强度
预计总训练量 / 预计耗时
[开始今日训练]
[跳过今天（顺延）]
[本次训练手动减载（减量） toggle]
```

视觉要求：

- 卡片高度建议 260–340px。
- 训练图片加暗色渐变遮罩，保证文字可读。
- 主按钮必须使用品牌主色。
- 次按钮为浅色描边或深色次级按钮。
- 手动减载开关位于主操作之后，不抢主 CTA。

按钮样式：

```css
.primary-button {
  height: 48px;
  border-radius: 14px;
  background: var(--color-primary);
  color: #111;
  font-weight: 700;
}

.secondary-button {
  height: 44px;
  border-radius: 14px;
  background: transparent;
  border: 1px solid var(--border);
  color: var(--text-secondary);
}
```

---

### 9.3 休息日卡片

休息日状态 B 应弱化运动冲动，强调恢复。

内容结构：

```text
今天是休息日
好好恢复，身体正在变强。

下次训练日
5月16日（周五）
上肢推（力量）

恢复建议
- 睡眠：保证 7–9 小时
- 营养：保持蛋白质摄入
- 活动：轻度拉伸或散步

[今日休息] disabled
[想加练？]
```

视觉：

- 可使用浅蓝图标或蓝色描边。
- 插画风格应简洁、扁平、低饱和。
- `今日休息` 为禁用态，不应看起来可点击。
- `想加练？` 为描边按钮，避免鼓励用户过度训练。

---

### 9.4 已完成状态卡片

完成状态 C 是正反馈模块，但仍保持克制。

内容结构：

```text
今日训练已完成 🎉
干得漂亮，继续保持！

训练组数 18
训练容量 8,640 kg
训练时长 62 分钟
预计消耗 612 kcal

[预设下次训练为减载期]
```

视觉：

- 使用浅绿色渐变或深绿色透明背景。
- 大勾图标可作为右侧视觉锚点。
- 不使用过多彩带、烟花等庆祝元素。
- 统计项横向 4 列，保持等宽。

### 9.5 身体状态模块

分为「录入中」与「已保存」两种状态。

#### 录入中

字段：

```text
体重 kg
腰围 cm
静息心率 bpm
睡眠时长 小时
主观疲劳度 1–10
```

控件要求：

- 数值输入框右对齐。
- 单位常驻显示，不应作为 placeholder。
- 疲劳度使用横向 slider，左低右高。
- 保存按钮靠近输入区右上或底部。

#### 已保存

指标卡：

```text
体重：72.6 kg
腰围：81.0 cm
静息心率：56 bpm
睡眠时长：7.2 小时
疲劳评分：3 / 10
BMI：23.1 正常
WHtR：0.44 优秀
```

视觉要求：

- 2–4 列自适应小卡。
- 正向变化用绿色小箭头。
- 不确定或需关注数据用橙色提示。
- BMI、WHtR 结果使用小胶囊标签，不使用大面积警告色。

---

### 9.6 营养对账模块

分为「录入中」与「已保存」两种状态。

#### 录入中

控件：

```text
训练日 / 休息日
克数直录 / 比例换算
碳水 g
蛋白质 g
脂肪 g
实时热量差值
```

规则：

- “训练日 / 休息日”为一级 segmented control。
- “克数直录 / 比例换算”为二级 segmented control。
- 差值为负数时显示“还可摄入”；正数时显示“超出”。
- 超出预算时使用橙色，不直接使用红色。

#### 已保存

展示：

```text
热量摄入：1,842 / 2,200 kcal
进度：84%

蛋白质：135 / 150 g，90%
碳水化合物：210 / 260 g，81%
脂肪：58 / 70 g，83%
```

进度条规范：

```text
高度：6px
圆角：999px
背景：低对比灰
蛋白质：绿色
碳水：黄色
脂肪：橙色
```

### 9.7 底部导航

推荐 4–5 个 Tab：

```text
今日
训练
营养 / 记录
数据
我的
```

当前设计中可使用：

```text
今日 / 训练 / 数据 / 我的
```

样式：

- 高度：72–84px
- 图标：20–22px
- 文案：10–11px
- 选中态：品牌主色图标 + 文案
- 未选中态：三级文本色
- 夜间模式可给选中图标轻微 glow

## 10. 图标与插画

### 10.1 图标

图标风格：

- 线性图标为主。
- 线宽 1.75–2px。
- 圆角端点。
- 不使用复杂填充图标。
- 重要状态可使用小面积实心图标。

常用图标：

```text
训练：哑铃 / 计时 / 清单
营养：餐具 / 圆环 / 加号
恢复：月亮 / 云 / 心率
数据：柱状图 / 趋势线
状态：勾 / 警告 / 铃铛
```

### 10.2 插画与图片

训练图：

- 用于训练卡主视觉。
- 建议使用真实健身照片或半写实渲染。
- 图片需加 40–60% 暗色渐变遮罩。
- 人物不要过度夸张，避免健美比赛风。

恢复插画：

- 使用扁平插画。
- 低饱和蓝 / 灰 / 绿。
- 画面表达休息、补水、拉伸、睡眠。

---

## 11. 动效规范

动效应服务于“状态变化”，不要炫技。

建议：

```text
页面切换：180–240ms ease-out
卡片进入：160–220ms，轻微上移 8px
按钮点击：scale 0.98，80–120ms
进度条加载：400–600ms
完成勾出现：300ms stroke animation
切换日夜模式：250–350ms crossfade + surface color transition
```

不建议：

- 大面积弹跳。
- 复杂 3D 翻转。
- 长时间 loading。
- 频繁呼吸光效。

---

## 12. 日夜模式映射规则

日间与夜间模式必须是同一套组件，不应像两个 App。

| 用途 | 日间模式 | 夜间模式 |
|---|---|---|
| 页面背景 | `#F7F8F2` | `#0F1416` |
| 主卡背景 | `#FFFFFF` | `#151B1E` |
| 次级卡背景 | `#F4F6EF` | `#1B2225` |
| 主文字 | `#1B1D1F` | `#F5F7F8` |
| 次级文字 | `#5F6468` | `#B6BEC4` |
| 弱文字 | `#9AA0A6` | `#7E878D` |
| 边框 | `#E6E9DF` | `#2B3438` |
| 主按钮 | `#B8E81E` | `#B8E81E` |
| 成功 | `#75D64B` | `#75D64B` |
| 警告 | `#F5A623` | `#F5A623` |

---

## 13. 可访问性要求

最低要求：

- 正文文字与背景对比度不低于 4.5:1。
- 小字不低于 11px。
- 主要按钮高度不低于 44px。
- 可点击元素最小热区 44px × 44px。
- 颜色不能作为唯一信息表达，需配合文字、图标或数值。
- 夜间模式避免纯白大面积文字，可使用 `#F5F7F8` 或 `#E9EEF0`。

---

## 14. 文案语气

文案应专业、简洁、行动明确。

推荐语气：

- “建议从下周一开启”
- “连续8周未减载，建议安排恢复周”
- “好好恢复，身体正在变强”
- “干得漂亮，继续保持”
- “还可摄入 358 kcal”
- “超出 220 kcal”

避免：

- 过度鸡血：“燃爆今天”“干翻自己”
- 医疗化恐吓：“严重损伤风险”
- 模糊表达：“可能不太好”
- 过多感叹号

## 17. 设计禁区

不要：

- 使用纯黑背景 + 纯白文字造成刺眼。
- 同时使用过多高饱和颜色。
- 将所有卡片都加绿色描边。
- 在同一屏放置过多复杂图表。
- 让图片主视觉抢过训练操作按钮。
- 使用过度拟物的金属、火焰、爆炸元素。
- 用红色大面积表达疲劳或超标。
- 将日间模式和夜间模式做成两套完全不同的布局。

## 18. 一句话总结

该视觉系统的核心是：**用柔和日间背景承载清晰计划，用深石墨夜间界面承载高对比数据，以黄绿色作为训练进步的统一视觉线索。**
